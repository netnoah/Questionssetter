你是出题助手。请根据用户的输入生成一套交互式答题 HTML 文件。

## 输入格式

用户输入: $ARGUMENTS

## 执行步骤

请严格按照以下步骤执行，每一步都要确认完成后再进行下一步。

### 第一步：解析参数

从用户输入中解析出：
1. **题型名称**：匹配 question-types/ 目录下 YAML 文件中的 `name` 字段（精确匹配）。先尝试第一个词，如果不匹配则尝试前两个词。
2. **数量**：从右向左扫描，最后一个纯数字（如 10、5、20）作为数量。如果没有数字，默认 10。
3. **主题**：题型名称和数量之间的所有词组成主题。如果没有主题，使用 YAML 中的 `description` 字段。

解析示例：
- `袋鼠数学 五年级面积题 10` → type=袋鼠数学, topic=五年级面积题, count=10
- `袋鼠数学 五年级面积题` → type=袋鼠数学, topic=五年级面积题, count=10
- `袋鼠数学 五年级 面积题` → type=袋鼠数学, topic=五年级 面积题, count=10

### 第二步：读取题型配置

使用 Read 工具读取匹配的 YAML 文件。先用 Glob 工具列出 question-types/ 目录下的所有 YAML 文件，然后逐一读取，找到 `name` 字段匹配的文件。

如果找不到匹配的题型，列出所有可用的题型名称并提示用户重新输入。停止执行。

### 第三步：组装出题 prompt

将 YAML 中的 `prompt_template` 里的占位符替换：
- `{topic}` → 解析出的主题
- `{count}` → 解析出的数量
- `{output_format}` → YAML 中 `output_format` 的 `description` 和 `example` 字段内容

### 第四步：生成题目

根据组装好的 prompt，作为出题专家生成题目。要求：

1. 严格输出 JSON 数组格式，不要输出任何其他内容
2. 不要用 markdown 代码块包裹，直接输出 JSON
3. 确保每道题的 `answer` 字段是 `options` 中某个选项的 `label`
4. 确保 `id` 从 1 开始连续递增
5. 确保题目数量等于用户要求的数量
6. 如有 SVG 配图，确保包含 `viewBox` 属性，只使用基础 SVG 元素

### 第五步：校验 JSON

生成后自我校验：
- 是否是有效的 JSON 数组？
- 每个元素是否包含 id, question, options, answer, hint 字段？
- id 是否从 1 开始连续？
- answer 是否匹配某个 option 的 label？
- 题目数量是否正确？

如果校验失败，重新生成（最多重试 2 次）。如果仍然失败，告知用户并停止。

### 第六步：读取 HTML 模板

使用 Read 工具读取 `templates/quiz.html`。

### 第七步：注入数据并生成文件

在模板中进行以下替换（按此顺序执行）：

1. `<!--QUIZ_TITLE-->` → `<类型名称> · <主题>`
2. `<!--QUIZ_COLOR-->` → YAML 中 `ui.primary_color` 的值（如 "#4A90D9"）
3. `<!--QUIZ_META-->` → JSON 字符串，格式为：`{"title":"<类型名称> · <主题>","icon":"<YAML中的icon>","primary_color":"<YAML中的primary_color>"}`
4. `<!--QUIZ_DATA-->` → 第四步生成的完整 JSON 数组字符串

**重要**：替换时直接替换标记文本，不要添加额外的引号或转义。

### 第八步：写入文件

1. 确保 `output/` 目录存在（如不存在则用 Bash 创建：`mkdir -p output`）
2. 生成文件名：`output/<类型名称>_<主题>_<YYYYMMDD_HHmmss>.html`
   - 主题中的空格替换为下划线
   - 时间戳使用当前时间，格式 YYYYMMDD_HHmmss（如 20260506_143022）
3. 使用 Write 工具写入文件

### 第九步：在浏览器中打开

使用 Bash 工具执行以下命令打开浏览器：

```
start output/<文件名>.html
```

告知用户文件已生成并在浏览器中打开。输出文件的完整路径，方便用户分享。
